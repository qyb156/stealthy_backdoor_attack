# Defence baselines (RQ4, Table 6)

Re-implementations of the three input-space defences evaluated against ASTT, run under the
unified environment of Section 5.5. Each script takes a JSONL of labelled candidate samples
(`{"code": "<source>", "label": 0|1}`, where `label=1` marks an actually poisoned sample)
and reports precision / recall of the defence.

| Script | Defence | Idea | Expected result vs. ASTT |
|---|---|---|---|
| `compiler_defense.py` | COMPILER | flag samples that fail to parse/compile | ~0 (ASTT outputs stay syntactically valid) |
| `onion_defense.py` | ONION | leave-one-out perplexity outlier tokens (CodeGPT) | very low (no unnatural input token inserted) |
| `codedetector_defense.py` | CodeDetector | integrated-gradient token attribution over the victim | very low (trigger is structural, not a salient token) |

## Usage

```bash
pip install -r ../requirements.txt

# COMPILER
python compiler_defense.py --input candidates.jsonl --lang java

# ONION (needs a pretrained code LM)
python onion_defense.py --input candidates.jsonl \
    --lm microsoft/CodeGPT-small-java --threshold 0.0 --trigger-tokens ret_var_

# CodeDetector (needs the trained victim classifier)
python codedetector_defense.py --input candidates.jsonl \
    --model_dir ../Defect-detection/code/saved_models --k 3.0 --trigger-tokens ret_var_
```

## Building the candidate set

`candidates.jsonl` is produced by mixing the poisoned samples generated by
`preprocess_atttack_strategy3_final_kmeans.py` (label 1) with an equal number of clean
samples (label 0) from the same split. For the dead-code comparison row of Table 6, replace
the ASTT trigger with the dead-code trigger `int ret_var_ = 1726;` when generating the
poisoned inputs.

> CodeDetector's original implementation is not public (Section 6.4); the version here is a
> faithful re-implementation on Captum's `IntegratedGradients` and is provided for
> transparency and to enable the quantitative comparison reported in Table 6.
